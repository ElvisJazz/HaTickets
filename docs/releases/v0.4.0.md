## 版本定位

`v0.4.0` 的核心目标是将 Mobile 模块的驱动后端从 Appium 完全迁移到 UIAutomator2 (u2) 直连，大幅降低部署门槛并显著提升抢票热路径性能。

## 主要功能提升

### 1) UIAutomator2 直连替代 Appium（核心变更）

- 驱动后端从 Appium（需 Node.js + Appium Server 常驻）迁移为 UIAutomator2 直连（仅需 adb）。
- 每次操作延迟从 ~100-200ms 降至 ~30-60ms，热路径一轮流程从 ~800ms-1.2s 压缩到 ~200-400ms。
- 完整适配层：`_find`、`_find_all`、`_click_coordinates`、`_element_rect` 等统一接口兼容 u2 元素类型。
- 新增 `_appium_selector_to_u2()` 选择器翻译，支持 UiSelector 语法解析（text/textContains/textMatches/className/clickable/instance）。

涉及文件：
- `mobile/damai_app.py`
- `mobile/config.py`
- `tests/unit/test_mobile_damai_app_u2_adapter.py`（新增 652 行）
- `tests/unit/test_mobile_config.py`

### 2) 热路径坐标缓存

- 新增 `_cached_tap()`：首次查找元素后缓存坐标，热重试时单次 HTTP 调用完成点击（冷路径 3 次 → 热路径 1 次）。
- 新增 `_cached_hot_path_coords` bot 级缓存，跨重试复用。
- `_wait_for_purchase_entry_result` 轮询优化：每次轮询从 4 个选择器（~240ms）减少到 1 个（~60ms）。

涉及文件：
- `mobile/damai_app.py`
- `mobile/hot_path_benchmark.py`

### 3) 搜索与导航鲁棒性增强

- `_qualify_resource_id()`：自动补全裸 ID（`img_jia` → `cn.damai:id/img_jia`），修复 u2 后端元素定位失败。
- 关键词重试前新增 popup dismiss + 页面状态恢复，避免弹窗阻塞后续搜索。
- 从不匹配详情页返回时主动 dismiss popup。
- `_fast_recover_to_detail` 改为逐步 back，避免跳过 detail_page。

涉及文件：

## 兼容性与行为变化（升级必读）

### 1) Appium 不再作为可用方案

- 所有文档和配置模板已移除 Appium 相关内容。
- 代码中 Appium 分支暂时保留（计划在后续版本清理），但不再推荐使用。

### 2) 配置字段变化

- `config.example.jsonc` 移除了 `driver_backend`、`server_url`、`device_name`、`serial` 字段。
- 用户只需填写 `udid`（`adb devices` 序列号）即可。

### 3) 部署简化

- 不再需要安装 Node.js、Appium 或启动 Appium Server。
- 前提条件仅：Python + Poetry + adb 可用 + Android 设备连接。

## 从 v0.3.0 升级到 v0.4.0（建议步骤）

1. 更新代码并安装依赖
   `poetry install`
2. 更新配置（如需要）
   `cp mobile/config.example.jsonc mobile/config.jsonc`
   只需填写 `udid` 和演出相关字段，无需 `server_url` 等 Appium 配置。
3. 先做安全探测（推荐）
   `./mobile/scripts/start_ticket_grabbing.sh --probe --yes`
4. 再执行正式抢票
   `./mobile/scripts/start_ticket_grabbing.sh --yes`

> 注意：不再需要启动 Appium Server，可以直接跳过原来的 `start_appium.sh` 步骤。

## 已知边界

- 线上风控与库存波动仍会影响最终结果，自动化只能提升效率，不能保证 100% 成功。
- u2 直连依赖设备上的 ATX agent，偶发断连时重新连接 USB 即可恢复。
- 页面结构更新可能导致定位失效，建议优先用 `--probe` 验证后再正式执行。
