# HaTickets v0.4.0（用户版发布说明）

发布日期：2026-04-01

这版最大的变化：**不再需要 Appium**。部署更简单，抢票更快。

## 你会直接感受到的变化

- **部署门槛大幅降低**：不再需要安装 Node.js 和 Appium Server，只要 `adb` 能识别手机就行。
- **抢票速度提升 2-3 倍**：驱动直连设备，每次点击操作从 ~150ms 降到 ~40ms。
- **配置更简单**：配置文件只需填 `udid` 和演出信息，不再有 `server_url`、`device_name` 等字段。
- **快速开始流程简化**：从 5 步减少到 4 步。

## 推荐使用流程（4 步）

1. 安装依赖
   `poetry install`
2. 连接手机，确认 `adb devices` 能识别
3. 先做安全探测（不提交订单）
   `./mobile/scripts/start_ticket_grabbing.sh --probe --yes`
4. 正式抢票
   `./mobile/scripts/start_ticket_grabbing.sh --yes`

> 没有"启动 Appium"这一步了。连上手机，直接跑。

## 自然语言入口（可选）

和 v0.3.0 一样，你可以用一句话生成配置并做探测：

```bash
./mobile/scripts/run_from_prompt.sh --mode probe --yes "给张三和李四抢4 月 6 号张杰的北京站演唱会内场门票，票价 1680 元"
```

## 从 v0.3.0 升级注意

- 如果你之前的 `config.jsonc` 里有 `server_url`、`driver_backend`、`device_name`，可以删掉这些字段，或者直接从模板重新生成：
  `cp mobile/config.example.jsonc mobile/config.jsonc`
- 不需要再运行 `./mobile/scripts/start_appium.sh`。

## 已知边界

- 线上库存、风控、验证码会影响结果，自动化不能保证 100% 成功。
- 建议每次正式抢票前，先跑一次 `--probe` 确认页面和配置都正常。
